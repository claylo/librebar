## Decisions made

- **Type-state builder pattern** — `Builder` → `ConfiguredBuilder<C>` when config is requested, each with its own `start()`. Achieves the spec's ideal API (`.config::<Config>().logging().start()`) without the plan's fallback `start_with_config::<C>()` method. See `src/lib.rs:196-334`.
- **`C: Serialize` bound on ConfigLoader::load()** — Required because `serde_json::to_value(C::default())` seeds the merge with struct defaults. The plan omitted this bound.
- **`serde-saphyr = "0.0"`** — Confirmed at `0.0.23` on crates.io. Semver range is correct.
- **No benchmarks in Phase 1** — Removed the `[[bench]]` section and divan dev-dep from the scaffold. Benchmarks are Phase 3.
- **hyper as sole HTTP stack** — Per design spec. No reqwest, no ureq.

## Landmines

- **`tracing_subscriber::registry().init()` is once-per-process** — Builder tests that call `.logging()` will conflict if run in the same process. Tests use `cargo nextest` (process-per-test) to avoid this. Do not switch to `cargo test`.
- **Test attribute ordering** — `#![allow(missing_docs)]` must come BEFORE `#![cfg(feature = "...")]` in integration test files, or clippy errors on the empty crate.
- **nextest filter syntax** — Use `binary(name)` to filter integration test files, not `test(name)` (which matches test function names). The Phase 1 plan had the wrong filter.
- **`rust-toolchain.toml`** — Pinned to `1.94.1`, which is the minimum required version (not arbitrary). The crate uses Rust 2024 edition features (let-chains in `config.rs`). Do not lower below `1.94.1`.
- **Uncommitted changes** — Two files have uncommitted changes on `main`: README Config example fix and design spec notes from `cli-batteries` research. These should be committed before starting Phase 2. Run `git diff` to see exact scope.
