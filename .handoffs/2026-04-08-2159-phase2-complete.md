## Decisions made

- **Composable layer stack** -- When both `logging` and `otel` are active, layers are collected into `Vec<Box<dyn Layer<Registry>>>` and composed in one `.with(layers)` call. When only `logging` is active, the direct `.with(filter).with(log_layer)` path is preserved (zero boxing overhead). See `src/lib.rs:252-277`.
- **`BoxedLayer` type alias in otel** -- `build_otel_layer()` returns `Option<BoxedLayer>` instead of a generic layer type. This avoids complex type-level composition issues when chaining `OpenTelemetryLayer` with `JsonLogLayer`. See `src/otel.rs:26-27`.
- **Shutdown stored in App, not global** -- Unlike cli-batteries which uses a global `Lazy<(Sender, Receiver)>`, rebar's `ShutdownHandle` lives in `App`. Avoids global state, allows multiple instances in tests. See `src/shutdown.rs`.
- **SIGTERM registration before spawn** -- `register_signals()` creates the SIGTERM `Signal` synchronously and propagates errors via `ShutdownInit`, then moves it into the spawned task. The plan had `expect()` inside the spawn which would swallow errors. See `src/shutdown.rs:64-67`.
- **tokio `macros` feature required** -- `tokio::select!` in the signal handler requires the `macros` feature. The plan omitted this; caught during code review. See `Cargo.toml:49`.
- **Crash timestamp duplication** -- `crash.rs` duplicates `format_timestamp()` and `days_to_ymd()` from `logging.rs`. Intentional: crash must be standalone with zero feature dependencies. Could extract to `src/time.rs` later.
- **Drop order is load-bearing** -- In `App`, `_otel_guard` is declared before `_logging_guard` so OTEL flushes spans while the logger is still active. Rust drops fields in declaration order. See `src/lib.rs:91-94`.
- **HTTP protobuf + hyper-client as default OTEL transport** -- Per design spec. `otel-grpc` feature adds tonic. Note: `opentelemetry-http` pulls reqwest as a transitive dep despite the "no reqwest" design goal -- unavoidable with the current OTEL stack.

## Landmines

- **`tracing_subscriber::registry().init()` is once-per-process** -- Tests that exercise the builder's logging or OTEL path will conflict if run in the same process. Use `cargo nextest run` (process-per-test). Do not switch to `cargo test`.
- **Test attribute ordering** -- `#![allow(missing_docs)]` must come BEFORE `#![cfg(feature = "...")]` in integration test files, or clippy errors on the empty crate.
- **Drop order in App struct** -- `_otel_guard` MUST be declared before `_logging_guard`. If someone reorders the struct fields, OTEL's final flush writes to a dead logger. No compile-time guard against this.
- **`rust-toolchain.toml`** -- Pinned to `1.94.1`. The crate uses Rust 2024 edition features (let-chains in `config.rs`). Do not lower below `1.94.1`.
- **OTEL env vars in tests** -- `otel_test.rs` clears `OTEL_EXPORTER_OTLP_ENDPOINT` before each test. If the host has this set (common in dev environments), tests that check "no endpoint = no layer" would fail without the cleanup.
- **reqwest transitive dep** -- `opentelemetry-otlp` with `http-proto` + `hyper-client` still pulls reqwest via `opentelemetry-http`. This contradicts the "no reqwest" design goal but is outside our control. Monitor for OTEL SDK releases that make this optional.
