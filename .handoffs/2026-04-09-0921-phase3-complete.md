## Decisions made

- **base64 as explicit dep for cache** — Not a transitive dep when `otel` is off. Added directly rather than reinventing with hex encoding.
- **`update` implies `cache`** — Update needs 24h caching to avoid hammering GitHub on every run. Composition via feature graph, not cfg-gating.
- **hyper-rustls for TLS** — Pure Rust TLS with Mozilla CA roots (`webpki-roots`), ring crypto provider. No system OpenSSL dependency. HTTP and HTTPS both work via `https_or_http()` connector.
- **gungraun alongside divan for bench** — Two feature-gated harnesses: `bench` (divan, wall-clock, any platform) and `bench-gungraun` (Valgrind/Callgrind instruction-count, Linux/Intel). Shared `bench` module via `#[cfg(any(...))]`.
- **Network tests gated by env var** — `REBAR_TEST_NETWORK=1` enables live HTTPS tests. Default test suite is offline-only.

## Landmines

- **`tracing_subscriber::registry().init()` is once-per-process** — Use `cargo nextest run` (process-per-test). `cargo test` will fail with "global default subscriber already set" errors.
- **Test attribute ordering** — `#![allow(missing_docs)]` must come BEFORE `#![cfg(feature = "...")]` in integration test files or clippy errors on the empty crate.
- **`unsafe_code` lint applies to test targets** — `[lints.rust] unsafe_code = "deny"` in `Cargo.toml` affects test crates too. Tests that mutate env vars need `#![allow(unsafe_code)]` (edition 2024 made `set_var`/`remove_var` unsafe).
- **hyper-rustls API requires explicit crypto provider** — `with_webpki_roots()` is gone in 0.27. Must use `with_provider_and_webpki_roots(rustls::crypto::ring::default_provider())`, which requires `rustls` as a direct dep (not just transitive).
- **Feature graph for `update`** — `update` implies `http` + `cache`. This means enabling `update` pulls in rustls, ring, hyper, base64, directories, serde_json. Consumers who want a lighter build should be aware.
